# Threat Moddeling Series
**By Stephen Reilly**

This is a series of blog posts in which I explore Threat Modelling, divided into four chapters:

| Post | Summary |
|------|---------|
| [E-commerce Platform](https://github.com/sreilly1977/dlh-cyber_security/blob/main/threat-modeling-fundamentals/0-ecommerce_platform.md) | A practical threat modeling exercise covering STRIDE analysis of checkout threats, identification of trust boundaries, and DREAD risk assessment for SQL injection vulnerabilities in an ecommerce platform architecture. |
| [Healthcare Mobile App](https://github.com/sreilly1977/dlh-cyber_security/blob/main/threat-modeling-fundamentals/1-healthcare_mobile_app.md) | A threat modeling exercise for a healthcare mobile app identifying unauthorized PHI exposure via IDOR, details STRIDE-based threats, and prioritizing MFA, encryption, and attribute-based access control as essential HIPAA-aligned mitigations. |
| [IoT Smart Thermostat](https://github.com/sreilly1977/dlh-cyber_security/blob/main/threat-modeling-fundamentals/2-iot_smart_thermostat.md) | A threat modeling exercise identifying critical IoT-specific vulnerabilities in a smart thermostat—such as insecure firmware updates and exposed debug interfaces—using the DREAD methodology to advocate for a hardware-rooted trust model with secure, zero-trust OTA architectures. |
| [Financial Trading Platform](https://github.com/sreilly1977/dlh-cyber_security/blob/main/threat-modeling-fundamentals/3-financial_trading_platform.md) | A defense-in-depth threat modeling strategy for a financial trading platform that prioritizes data integrity over availability, employs the DREAD methodology to rank risks, and quantifies how layered controls can achieve approximately 99.4% cumulative risk reduction while maintaining critical low-latency performance requirements. |
| [Cloud Storage Service](https://github.com/sreilly1977/dlh-cyber_security/blob/main/threat-modeling-fundamentals/4-cloud_storage_service.md) | A critical threat modeling for a cloud storage service, identifying key risks such as compromised authentication flows, encryption key management failures, and malware injection via file uploads. |
